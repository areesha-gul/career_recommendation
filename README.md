## Overview

An end-to-end AI system that predicts MBTI personality type from questionnaire/text input and recommends personalised career paths using a five-module pipeline.

This project uses a hybrid data setup:

- MBTI500 text data for personality classification experiments (default)
- Synthetic questionnaire data generated by `generate_data.py` for the structured demo flow
- Curated career and skill data for recommendation and constraint matching

| Module | Technique | Purpose |
|--------|-----------|---------|
| M1 | NLTK + Multinomial Naive Bayes | Text preprocessing & dimension scoring |
| M2 | Decision Tree C4.5 | MBTI personality classification |
| M3 | CSP with AC-3 + Backtracking | Career matching under constraints |
| M4 | A* Informed Search | Career path finding on graph |
| M5 | Hill Climbing | Recommendation ranking optimisation |

## Setup

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Generate the synthetic career data
python generate_data.py

# 3. Ensure MBTI500 is available in data/raw/ (MBTI500.csv)

# 4. Train MBTI models (text + questionnaire)
python train_model.py

# 5. Launch the Streamlit interface
streamlit run app.py
```

If you want to train text on the original Kaggle mbti_1.csv instead:

```bash
python train_model.py --text-source kaggle
```

## Data Strategy

Use the two datasets for different jobs:

- Synthetic data from `generate_data.py` for the questionnaire-based pipeline and integration testing
- MBTI500 data (default) or Kaggle mbti_1.csv for the text classification side of the project
- Career and skill JSON files for CSP, A*, and ranking logic

This keeps the project consistent while letting you claim both realistic text data and controlled structured data in the methodology.

## Model Evaluation

Train and evaluate using MBTI500 (default text source):

```bash
python train_model.py
```

This command uses all available rows from the selected source, then performs a stratified train/test split for evaluation.

Generated evaluation artifacts:

- `models/training_metrics.json` (accuracy, macro-F1, weighted-F1, optional CV summary)
- `models/classification_report.json` (per-class precision/recall/F1)
- `models/confusion_matrix.csv` (16x16 confusion matrix)
- `models/mbti_text_dt.pkl` (trained pipeline)

Optional commands:

```bash
# disable cross-validation (faster)
python train_model.py --cv-folds 0

# use original Kaggle mbti_1.csv instead of MBTI500
python train_model.py --text-source kaggle

# combine MBTI500 + Kaggle when both are present
python train_model.py --text-source both
```
